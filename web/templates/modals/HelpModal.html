Help

Usage

Every field, by default, will be mapped as string.
If you want to force a string for the config value, you can wrap the value with double quotes.

  • "my_string"
  • my_string

Example

"hello world"
 
{
    config_key: "hello world"
}
 
config_key: hello world
 
[
    'config_key' => 'hello world'
]

Usage

Prefix the value with a # character.

  • #34
  • #20.5

Example

#20.5
 
{
    config_key: 20.5
}
 
config_key: 20.5
 
[
    'config_key' => 20.5
]

Usage

Simply use the true and false keywords.

  • true
  • false

Example

true
 
{
    config_key: true
}
 
config_key: true
 
[
    'config_key' => true
]

Usage

Use the square bracket form to represent an array.
You can use nested arrays if needed

For now, dictionaries/associative arrays aren't supported.

PHP exports: If you use PHP exports with define statements, the config value will be returned as serialized arrays (using native PHP serialization mechanism) since PHP's define() function doesn't support array as values.

  • [my_string_value, my_other_string-value]
  • [#14, #56]
  • [[#14, #56], ["my_val", "another_val"]]

Example

[[#14, #56], ["my_val", "another_val"]]
 
{
    config_key: [
        [
            14,
            56
        ],
        [
            "my_val",
            "another_val"
        ]
    ]
}
 
config_key:
    -
        - 14
        - 56
    -
        - my_val
        - another_val

[
    'config_key' => [
        [
            14,
            56
        ],
        [
            "my_val",
            "another_val"
        ]
    ]
]

Usage

Use the null word. The case is insensitive (i.e. both null and NULL are accepted).

  • null
  • NULL

Example

null
 
{
    config_key: null
}
 
config_key: null
 
[
    'config_key' => null
]
